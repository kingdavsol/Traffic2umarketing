## Session Summary

This session fixed several critical issues with the QuickSell application:

1. **Registration was blocked** - CAPTCHA middleware was blocking registration when reCAPTCHA keys weren't configured
2. **CORS blocking mobile apps** - Mobile apps couldn't connect due to strict CORS policy
3. **Missing user fields in API response** - Frontend expected fields not returned by backend

---

## Issues Fixed

### 1. Registration Blocked by CAPTCHA (CRITICAL)

**Problem:** The CAPTCHA middleware was returning a 500 error in production when `RECAPTCHA_SECRET_KEY` wasn't configured, completely blocking user registration.

**Fix:** Modified `backend/src/middleware/security.ts` to skip CAPTCHA verification with a warning instead of blocking:
```typescript
if (!recaptchaSecret) {
  if (process.env.NODE_ENV === 'production') {
    logger.warn('RECAPTCHA_SECRET_KEY not configured - CAPTCHA verification skipped.');
  }
  return next();  // Allow through instead of blocking
}
```

### 2. CORS Blocking Mobile Apps

**Problem:** The CORS configuration only allowed specific origins, blocking requests from mobile apps (which don't send an origin header).

**Fix:** Updated `backend/src/server.ts` CORS configuration:
```typescript
app.use(cors({
  origin: (origin, callback) => {
    // Allow requests with no origin (mobile apps, Postman, curl)
    if (!origin) return callback(null, true);
    // Allow configured origins
    if (allowedOrigins.includes(origin)) return callback(null, true);
    // Allow Expo/React Native development
    if (origin.startsWith('exp://') || origin.includes('expo')) return callback(null, true);
    // Block other origins in production
    callback(new Error('Not allowed by CORS'));
  },
  // ...
}));
```

### 3. Missing User Fields in Login/Register Response

**Problem:** Frontend expected `subscriptionTier`, `points`, and `level` fields in the user object, but backend wasn't returning them.

**Fix:** Updated `backend/src/controllers/authController.ts` to include all required fields:
```typescript
user: {
  id: user.id,
  email: user.email,
  username: user.username,
  subscriptionTier: user.subscription_tier || 'free',
  points: user.points || 0,
  level: user.level || 1,
  isAdmin: user.is_admin || false,
}
```

## Remaining Configuration (Optional)

To enable CAPTCHA protection for spam prevention, configure these environment variables on the VPS:

```bash
# Add to /var/www/quicksell.monster/.env
RECAPTCHA_SITE_KEY=<from Google reCAPTCHA admin>
RECAPTCHA_SECRET_KEY=<from Google reCAPTCHA admin>
```

Then update the frontend to include CAPTCHA tokens in registration requests.
