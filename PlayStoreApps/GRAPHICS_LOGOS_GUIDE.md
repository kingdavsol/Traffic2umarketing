## How to Create Graphics

### Option 1: AI Design Tools (Free - Quick)

1. **Canva.com**
   - Sign up free
   - Search "Google Play Store Icon"
   - Customize template with app colors
   - Download PNG

2. **Figma.com**
   - Freemium tool
   - Better for professional design
   - More control over assets

3. **Adobe Express** (formerly Adobe Spark)
   - Free tier available
   - Easy to use
   - Smartphone optimized

### Option 2: Hire Designer ($100-500 per app)

Platforms:
- Fiverr.com - Budget friendly ($100-200)
- Upwork.com - Vetted professionals ($300-500)
- 99designs.com - Higher quality ($500+)

Brief Template for Designer:
```
App: [Name]
Primary Color: [Hex]
Secondary Color: [Hex]
Style: Modern, minimalist, professional
Icon Concept: [Description]
Target Audience: [Description]
Deliverables: 512x512 icon, 1024x500 banner, 5 screenshots
Budget: $[Amount]
Deadline: [Date]
```

### Option 3: AI Generation Tools ($10-50)

1. **Midjourney.com**
   - Text to image
   - High quality
   - Great for banners

2. **DALL-E** (via OpenAI)
   - Easy to use
   - Fast generation
   - Lower cost

3. **Stable Diffusion**
   - Free option
   - Open source
   - Good quality

Example Prompts:
```
"Professional app icon for Mental Health app,
purple and pink gradient, meditation theme,
clean modern design, 512x512"

"Google Play Store banner for postpartum fitness app,
new mother doing gentle yoga, soft lighting,
wellness aesthetic, 1024x500"
```

## Play Store Asset Requirements

### Icon Requirements:
- Size: 512x512 pixels
- Format: PNG with alpha
- Must fit in 48dp (with padding)
- No rounded corners (system rounds them)
- Avoid small details (scale down to 48dp test)

### Banner Requirements:
- Size: 1024x500 pixels
- Format: PNG, JPG, or GIF
- Aspect ratio: 2.04:1
- No text overlay (or minimal)
- Eye-catching, professional

### Screenshot Requirements:
- Minimum: 2, Recommended: 5
- Size: 1440x2560 or 1080x1920
- Format: PNG or JPG
- Aspect ratio: 9:16
- Include text captions (optional)
- Show key features
- Device frames (optional)

---

## Design Tips

### App Icons:
1. Make them bold and recognizable at small sizes
2. Use app's primary color
3. Keep designs simple (will be 48dp on home screen)
4. Test at multiple sizes (512, 192, 144, 96, 72, 48 dp)
5. Avoid copyrighted symbols

### Banners:
1. Feature actual app usage/benefit
2. Include 1-2 people for relatability
3. Use app's color scheme
4. Text should be minimal (if any)
5. High quality photography
6. Make sure it works at all sizes

### Screenshots:
1. Show actual app interface (not mockups)
2. Highlight key features, not all features
3. Use consistent styling (same device frame)
4. Add captions explaining benefits
5. Show the benefit, not the feature
6. Test on actual devices
