## Domain Allocation Strategies

### Strategy 1: Individual Domains (Recommended for Branding)
```
Each app gets its own primary domain
Cost: ~$10-15/year per domain × 20 = $200-300/year
Benefit: Maximum branding, SEO per app, independent reputation
Example:
  mentalhealth-pro.com
  postpartum-fitness.com
  adhd-manage.com
  ... (17 more)
```

### Strategy 2: Subdomains Under Parent Domain (Cost-Effective)
```
Single parent domain with 20 subdomains
Cost: ~$10-15/year for parent domain
Benefit: Centralized management, lower cost, related branding
Example:
  app1.example.com    → Mental Health Pro
  app2.example.com    → Postpartum Fitness
  app3.example.com    → Local Services
  ... (17 more)

Parent dashboard: example.com
```

### Strategy 3: Branded Subdomains Under Parent Domain
```
Parent domain with branded subdomains
Cost: ~$10-15/year for parent domain
Benefit: Balance between branding and cost
Example:
  mental-health.example.com
  postpartum.example.com
  local-services.example.com
  ... (17 more)
```

### Strategy 4: Hybrid Approach (Recommended for Scale)
```
Primary apps: Individual domains
Secondary apps: Subdomains under parent
Cost: ~$100-200/year
Benefit: Flexibility, scalability, targeted branding

Primary (Individual Domains):
  mentalhealth-pro.com         → App #1
  habit-tracker.app            → App #12
  ai-stylist.com               → App #13
  interview-prep.com           → App #16

Secondary (Subdomains):
  app2.example.com             → App #2
  app3.example.com             → App #3
  ... (14 more)
```

## Quick Domain Setup Command Reference

### Register Multiple Domains (Example with GoDaddy API)
```bash
# If using GoDaddy API for bulk registration:
curl -X POST https://api.godaddy.com/v1/domains/purchase \
  -H "Authorization: sso-key YOUR_KEY:YOUR_SECRET" \
  -H "Content-Type: application/json" \
  -d '{
    "domain": "mentalhealth-pro.com",
    "consent": {"agreedAt": "'$(date -Iseconds)'"}
  }'
```

### Update NEXTAUTH_URL Across All Apps
```bash
# Bash script to update all apps with production domain
for app_dir in PlayStoreApps/apps/*/; do
  app_name=$(basename "$app_dir" | sed 's/[0-9]*-//g')
  domain=$(echo "$app_name" | sed 's/-/\\-/g')

  if [ -f "$app_dir.env.production" ]; then
    sed -i "s|NEXTAUTH_URL=.*|NEXTAUTH_URL=https://$domain.com|g" "$app_dir.env.production"
  fi
done
```
