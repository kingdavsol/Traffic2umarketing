## 1. Security Hardening

### Critical Vulnerabilities Fixed

| Issue | Solution |
|-------|----------|
| No CAPTCHA on registration | Added reCAPTCHA v3 middleware to `/register` |
| No CAPTCHA on password reset | Added reCAPTCHA v3 middleware to `/forgot-password` |
| Hardcoded JWT secret fallback | Now throws error in production if not configured |
| Hardcoded encryption key | Now throws error in production if not configured |
| 50MB request body limit | Reduced to 10MB to prevent DoS |
| No account lockout | 5 failed attempts = 15 minute lockout |
| No HTTPS enforcement | Added middleware to reject HTTP in production |
| Missing security headers | Added HSTS, X-Frame-Options, CSP headers |

### New Security Middleware (`backend/src/middleware/security.ts`)

```typescript
// Functions available:
accountLockout        // Prevents brute force attacks
recordFailedAttempt   // Tracks failed login attempts
clearLoginAttempts    // Clears on successful login
verifyCaptcha         // reCAPTCHA v3 verification
enforceHttps          // Rejects HTTP requests in production
securityHeaders       // Sets comprehensive security headers
validateJwtSecret     // Validates JWT secret at startup
```

### Required Environment Variables for Production

```env
# CRITICAL - Must be unique and secure
JWT_SECRET=<generate with: node -e "console.log(require('crypto').randomBytes(32).toString('hex'))">
MARKETPLACE_ENCRYPTION_KEY=<generate with same command>

# Required for CAPTCHA
RECAPTCHA_SITE_KEY=<from Google reCAPTCHA admin>
RECAPTCHA_SECRET_KEY=<from Google reCAPTCHA admin>
```

### Frontend Integration Required

For CAPTCHA to work, frontend must include token in requests:
```javascript
const token = await grecaptcha.execute('SITE_KEY', {action: 'register'});
fetch('/api/v1/auth/register', {
  body: JSON.stringify({ ...formData, captchaToken: token })
});
```
