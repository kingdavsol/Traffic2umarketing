## Security Middleware Added (`backend/src/middleware/security.ts`)

### Functions
1. **`accountLockout`** - Middleware to prevent brute force attacks
2. **`recordFailedAttempt`** - Records failed login attempts
3. **`clearLoginAttempts`** - Clears attempts on successful login
4. **`verifyCaptcha`** - reCAPTCHA v3 verification middleware
5. **`enforceHttps`** - Rejects HTTP requests in production
6. **`securityHeaders`** - Sets comprehensive security headers
7. **`validateJwtSecret`** - Validates JWT secret at startup

### Security Headers Applied
- `Strict-Transport-Security` (HSTS)
- `X-Frame-Options: DENY`
- `X-Content-Type-Options: nosniff`
- `X-XSS-Protection: 1; mode=block`
- `Referrer-Policy: strict-origin-when-cross-origin`
- `Permissions-Policy` (blocks geolocation, mic, camera)

## Frontend Integration Required

For CAPTCHA to work, the frontend must:

1. Load reCAPTCHA v3 script:
```html
<script src="https://www.google.com/recaptcha/api.js?render=YOUR_SITE_KEY"></script>
```

2. Include `captchaToken` in registration/password-reset requests:
```javascript
const token = await grecaptcha.execute('YOUR_SITE_KEY', {action: 'register'});
const response = await fetch('/api/v1/auth/register', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ ...formData, captchaToken: token })
});
```
